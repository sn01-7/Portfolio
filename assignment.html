Project Overview

Objective: Compress text data using Huffman Coding to minimize storage requirements and enable efficient data transmission.

Huffman Coding is a lossless data compression algorithm that assigns variable-length codes to characters based on their frequencies in the text.

Task

You are required to implement a program that compresses a given text document using Huffman Coding. The program should also be able to decode the compressed file to retrieve the original text.

Code Snippet

Detailed Explanation of Huffman Coding

This process consists of several steps:

1. Analyze Character Frequency

Identify how often each character appears in the file. For example:

If the file contains the text "ABBCCCDDDDEEEEEE", frequencies would be:
  • A: 1
  • B: 2
  • C: 3
  • D: 4
  • E: 5

2. Build a Priority Queue of Nodes

Store each character with its frequency in a priority queue, where lower frequency items are at the top.

3. Construct the Huffman Tree

Create a binary tree that assigns shorter paths to frequently occurring characters:

                  *
                 / \
                *   E(5)
               / \
              D(4) *
                  / \
                 B(2) C(3)

4. Assign Binary Codes

Each character is assigned a binary code based on its path in the tree:

  • E: 1
  • D: 01
  • C: 001
  • B: 0001
  • A: 0000

5. Encode the Data

Using the binary codes, the original text is converted to an encoded version.

6. Save the Encoded Data & Tree

Both the binary data and Huffman Tree representation are stored to allow decompression.

7. Decompress the File

Using the Huffman Tree, the encoded binary data is decoded back to the original text.

Benefits of Huffman Coding

  • Efficiency: Reduces file size by assigning shorter codes to frequent characters.
  • Wide Application: Used in many compression standards such as ZIP and JPEG.
